Class Tc.Application

Responsible for application-wide issues such as the creation of modules.

Constructor

Tc.Application ( )

Properties

$ctx - jQuery

The jQuery context.

classes - object

A module can have three types of classes: - .mod Indicates that it is a base module (default -> no javascript need to be involved). Must occur excactly once. - .mod (e.g. .modBasic) Indicates that it is a module of type basic, which is derived from the base module. It can occur at most once. - .skin (e.g. .skinBasicSubmarine) Indicates that the module basic has the submarine skin. It will be decorated by the skin js (if existing). It can occur arbitrarily. Additionally, a module can have one type of data attributes: - data-connectors A comma-separated value containing the connector ids, the schema of a connector id is: e.g. MasterSlave1Master: name = MasterSlave, id = 1, role = Master The above indicates that the module should notify the MasterSlave connector (the mediator) over all state changes. The connector id is used to chain the appropriate modules together and to improve the reusability of the connector. It can contain multiple connector ids (e.g. 1,2,MasterSlave1Master) TODO: Is this still correct? The above statement referenced type 1 for data-connectors

config - Object

The configuration.

connectors - Object

Contains references to all modules on the page. Could be useful for example when there are interactions between Flash <-> JS.

The sandbox to get the resources from This sandbox is shared between all modules.
